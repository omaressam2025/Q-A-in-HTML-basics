100 سؤال وجواب في HTML (الأساسيات)

1- ايه هو HTML؟

HTML أو HyperText Markup Language هي اللغة الأساسية اللي بيتم استخدامها لبناء هيكل صفحات الويب. الهدف منها إنها تنظم المحتوى وتحدد شكله الأساسي زي العناوين، الفقرات، الصور، الجداول، والروابط. HTML مش لغة برمجة بالمعنى التقليدي، لأنها مش بتتعامل مع منطق أو شروط أو loops، لكنها لغة ترميز (Markup) يعني بتوضح للمتصفح كل جزء في الصفحة معناه إيه عشان يعرضه صح.

2- ايه معنى tag في HTML؟

tag أو الوسم هو الكود اللي بيتكتب بين أقواس < >، واللي بيحدد نوع العنصر. معظم الوسوم ليها وسم فتح (زي <p>) ووسم قفل (زي </p>). الوسوم دي بتقول للمتصفح ده نص، ده صورة، ده جدول وهكذا. بعض الوسوم ذاتية الإغلاق زي <img> و <br>.

3- ايه الفرق بين <h1> و <h6>؟

وسوم العناوين في HTML بتبدأ من <h1> لحد <h6>. <h1> أكبر عنوان وبيستخدم غالبًا للعنوان الرئيسي للصفحة. <h6> أصغر عنوان وبيستخدم للتقسيمات البسيطة أو الأقل أهمية. الفرق مش مجرد حجم الخط، لكن كمان أهمية في تحسين محركات البحث (SEO)، لأن العناوين بتوضح هيكل الصفحة لمحركات البحث.

4- ايه فائدة <p>؟

<p> هو وسم الفقرة (paragraph). بيستخدم لكتابة نصوص عادية داخل الصفحة. المتصفح تلقائيًا بيضيف مسافات قبل وبعد كل فقرة عشان يفصلها عن باقي النصوص، وده بيسهل القراءة.

5- ايه الفرق بين <br> و <p>؟

<br> بيعمل كسر سطر واحد بس، يعني بيكمل النص من السطر الجديد. <p> بيعمل فقرة كاملة مستقلة بتكون منفصلة بمسافات فوق وتحت. <br> مناسب للحالات اللي عايز فيها سطر جديد جوه نفس الفقرة زي كتابة شعر أو عناوين فيها أكثر من سطر.

6- ازاي أكتب تعليق في HTML؟

التعليقات في HTML بتكتب كده: <!-- تعليق هنا -->. التعليقات مش بتظهر للمستخدم العادي في الصفحة لكن بتبقى موجودة في الكود عشان المبرمجين يفهموا أو يوضحوا أجزاء من الكود.

7- ايه الفرق بين <head> و <body>؟

<head> بيحتوي على البيانات الوصفية للصفحة (metadata) زي العنوان <title>، ملفات CSS المربوطة، ملفات JavaScript الخارجية، ووسوم meta الخاصة بالوصف والكلمات المفتاحية. <body> هو الجزء اللي بيظهر فعليًا للمستخدم وبيحتوي المحتوى زي النصوص والصور والفيديوهات.

8- ايه هو attribute في HTML؟

attribute هو خاصية بتتكتب جوه وسم HTML عشان تضيف له معلومة أو وظيفة إضافية. زي <a href=\"link\"> اللي بتحدد الرابط اللي هيوديه العنصر. أو <img src=\"image.jpg\" alt=\"description\"> اللي بتحدد مصدر الصورة والنص البديل.

9- ايه الفرق بين relative و absolute URL في الروابط؟

absolute URL بيحتوي العنوان الكامل بما في ذلك البروتوكول (https://example.com/page.html). relative URL بيكون نسبي لمكان الملف الحالي. لو انت في مجلد معين وعايز تربط ملف تاني في نفس المجلد، تكتب بس اسمه (page.html) أو تطلع مجلد بمسار ../page.html. relative أسرع وأبسط لو الملفات على نفس السيرفر.

10- ازاي بضيف صورة؟

باستخدام وسم <img> اللي هو وسم ذاتي الإغلاق. أهم attributes فيه: src لمسار الصورة، و alt للنص البديل اللي بيوصف الصورة لو مش اتعرضت. مثال: <img src=\"photo.jpg\" alt=\"صورة طبيعية\">.

11- ايه هو alt attribute في <img>؟

alt attribute بيستخدم لكتابة وصف نصي للصورة. لو الصورة ما ظهرتش لأي سبب، النص البديل بيتعرض مكانها. كمان النص ده بيساعد محركات البحث إنها تفهم محتوى الصورة، وكمان بيساعد الأشخاص ضعاف البصر اللي بيستخدموا برامج قراءة الشاشة.

12- ايه الفرق بين <ul> و <ol>؟

<ul> هو unordered list يعني قائمة نقطية. <ol> هو ordered list يعني قائمة مرقمة. جوه القائمتين بتستخدم <li> لكتابة عناصر القائمة. الفرق إن ul بيظهر نقاط أو رموز، و ol بيظهر أرقام أو حروف حسب النوع.

13- ازاي بضيف لينك في HTML؟

باستخدام وسم <a>. لازم يكون فيه attribute اسمه href بيحدد الرابط. مثال: <a href=\"https://example.com\" target=\"_blank\">زور الموقع</a>. هنا target=\"_blank\" معناها افتح الرابط في نافذة جديدة.

14- ايه هو semantic HTML؟

semantic HTML يعني استخدام الوسوم اللي ليها معنى واضح وبتوضح وظيفة العنصر. زي <header>, <footer>, <article>, <section>. ده بيساعد محركات البحث وأدوات الوصول إنها تفهم المحتوى بشكل أوضح، وبيخلي الكود منظم أكتر.

15- ايه الفرق بين inline و block elements؟

block elements (زي <div>, <p>) بتبدأ في سطر جديد وتاخد العرض كله. inline elements (زي <span>, <a>) بتكمل في نفس السطر ومش بتكسر النص. الفرق ده مهم عشان تتحكم في توزيع العناصر في الصفحة.

16- ازاي بكتب جدول بسيط (<table>)؟

الجدول في HTML بيتبنى بوسم <table>، جواه بتحط صفوف <tr> وكل صف جواه خلايا: خلايا عنوان <th> (عادة في أول صف) وخلايا بيانات <td>. مثال كامل وظيفي: 

<table border="1"> <thead> <tr><th>الاسم</th><th>العمر</th></tr> </thead> <tbody> <tr><td>أحمد</td><td>25</td></tr> <tr><td>منى</td><td>30</td></tr> </tbody> </table>

نقاط مهمة: استعمل <thead>/<tbody> لتنظيم الجدول، وحط scope أو id/ headers لو الجدول معقّد لتحسين الوصولية.

17- ازاي بستخدم colspan و rowspan لدمج خلايا الجدول؟

لو عايز خلية تمتد على أكثر من عمود استخدم colspan="N"، ولو تمتد على صفوف استخدم rowspan="N". مثال: 

<tr> <td colspan="2">اسم كامل</td> </tr>

خلي بالك من تخطيط الصفوف والأعمدة بعد الدمج عشان ميتلغبطش ترتيب الخلايا، واختبر على شاشات مختلفة لو جدولك معقّد.

18- ايه أفضل ممارسات لكتابة جداول قابلة للوصول (accessible tables)؟

عشان الجدول يبقى سهل الوصول للي بيستخدموا screen readers: استخدم <th> مع scope="col" أو scope="row"، ضيف caption لشرح محتوى الجدول، استخدم headers وid لو layout معقّد، وتجنب استخدام الجداول للتخطيط البصري (layout) — الجداول لازم للبيانات فقط.

19- ازاي أعمل رابط داخلي (anchor) يودّي لمكان معين في نفس الصفحة؟

تستخدم خاصية id على العنصر الهدف، وبعدين تربطله بـ #id في href. مثال: 

<h2 id="section1">القسم الأول</h2> <a href="#section1">روح للقسم الأول</a>

دي طريقة بسيطة للسّكرول السلس لو ضفت CSS/JS، أو للملاحة السريعة داخل الصفحة.

20- ازاي أعمل صورة responsive عشان تناسب شاشات مختلفة؟

في HTML عندك أدوات جاهزة زي <img srcset="..." sizes="..."> و<picture>. الفكرة إنك تقدّم نسخ متعدّدة من الصورة (أحجام أو crops مختلفة) والمتصفح يختار الانسب حسب عرض الـ viewport وdevice pixel ratio. مثال مبسّط:

<img src="small.jpg" srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w" sizes="(max-width:600px) 480px, (max-width:1200px) 800px, 1200px" alt="وصف الصورة">

دا بيقلل تحميل صور كبيرة على الموبايل ويحسّن الأداء.

21- ايه هو <picture> وإمتى استخدمه بدل <img>؟

<picture> بيسمحلك تحدد <source> لكل media query أو نوع ملف (webp vs jpg) وتتحكّم في art direction (تغيّر crop أو مقطع الصورة على breakpoints مختلفة). استخدمه لو عايز بدائل مختلفة للصورة اعتمادًا على العرض أو دعم صيغ حديثة.

22- ازاي أضيف فيديو في HTML؟ وازاي أضمن توافقه مع Browsers؟

عنصر الفيديو: <video controls><source src="video.mp4" type="video/mp4"></video>. للحماية من مشاكل التوافق قدّم أكثر من source بصيغ مختلفة (mp4, webm). أضف attribute مثل controls لعرض أزرار التشغيل، poster لصورة معاينة، و muted أو autoplay حسب الحاجة (لكن Autoplay غالبًا تحتاج muted في بعض المتصفحات).

23- ايه أهمية نص alt للصورة من ناحية الوصولية وSEO؟

نص alt بيساعد الأشخاص اللي ما بيشوفوش الصورة (باستخدام screen readers) لأن البرنامج هيقرألك النص. كمان لو الصورة ما اتحمّلتش يظهر الـalt كبديل. لمحركات البحث، الـalt بيساعدها تعرف محتوى الصورة وبالتالي ممكن يظهر في بحث الصور—فهو مهم للأداء والـaccessibility.

24- ازاي أعمل رابط يفتح صفحة جديدة وآمن (target="_blank") بدون مشاكل أمان؟

لو هتستخدم target="_blank" ضيف كمان rel="noopener noreferrer" لتمنع الصفحة الجديدة من الوصول للـ window.opener وقلل مخاطر الـ tabnabbing:

<a href="https://example.com" target="_blank" rel="noopener noreferrer">افتح</a>

دي ممارسة أمان بسيطة وفعالة.

25- ايه الفرق بين inline element و block element عمليًا؟ ومتى اغير الـ display؟

الفرق الأساسي إن block يأخذ سطر جديد ويستطيع احتواء عناصر اخرى، بينما inline يبقى في نفس السطر ويأخذ قد المحتوى. لو محتاج تغير السلوك البصري ممكن تستخدم CSS: display:block أو display:inline-block. مثلاً لو عايز زرار يتصرف كعنصر inline لكن له أبعاد تستخدم inline-block.

26- ازاي أستخدم <form> مع validation بسيطة في HTML؟

HTML بيوفر attributes جاهزة زي required, type="email", min, max, pattern (لـ regex). المتصفح هيعرض رسائل افتراضية للمستخدم. مثال: 

<form> <input type="email" name="email" required> <input type="number" name="age" min="1" max="120"> <input type="submit" value="إرسال"> </form>

لكن لو عايز تحكم كامل في الرسائل أو الـ UX استخدم JavaScript لعمل validation مخصّص، مع الحفاظ على التحقق server-side.

27- ازاي أستخدم label بطريقة صحيحة لربطها بالـ input؟ في طريقتين: 1) تحط الـ input جوا الـ label، أو 2) تربطهم بالـ for وid. المثال الثاني مفيد لو بتحط ال label بعيد عن ال input. الربط مهم عشان لما المستخدم يكليك على الـ label التركيز يروح للـ input—وده مهم جدا للوصولية.

28- ايه الفرق بين placeholder و label؟ ولمتين استخدم كل واحد؟

الـ label لازم يظهر دايما ويشرح الحقل، أما placeholder فده نص مساعد مؤقت جوا الحقل ويختفي لما المستخدم يبدأ يكتب. من الأفضل دايما استخدام label ظاهرة علشان الـ accessibility، واستخدم placeholder للـ hint أو المثال.

29- ازاي أعمل input للـ email والـ tel والـ url؟ وليه تستخدمهم مش input type="text"؟

HTML بتقدّم أنواع مخصّصة: <input type="email">, type="tel", type="url". المتصفح يقدّم validation مبدئي ويعرض لوحة مفاتيح مناسبة على الموبايل (مثلاً لوحة فيها @ لtype="email"). ده يحسّن UX ويقلل أخطاء الإدخال.

30- ازاي أضمن إن الفورم يشتغل حتى لو جاڤاسكربت معطّل؟

اعتمد على progressive enhancement: خلي الـ form يملك action و method للسيرفر بحيث لو الجافاسكربت اتعطل السيرفر يتعامل مع الطلب. استخدم JavaScript كتحسين للـ UX مش كشرط وحيد لإرسال البيانات.

31- ازاي أستخدم input type="file" مع accept و multiple؟

تقدر تحط accept="image/*,.pdf" لتقييد أنواع الملفات، وmultiple عشان تسمح باختيار أكتر من ملف. مثال:

<input type="file" accept="image/*" multiple>

لكن تحقق دايمًا server-side لأن المستخدم ممكن يلغي القيود client-side.

32- ايه هو fieldset و legend؟ وليه نستخدمهم؟

fieldset يجمع مجموعة من عناصر الفورم تحت إطار واحد، وlegend بيوصف الإطار ده. ده مفيد لتنظيم الفورمات الكبيرة وتحسين الوصولية لأن screen readers بيفهموا إن الحقول دي مجموعة.

33- ازاي أتحكّم في ترتيب التبويب (tab order) باستخدام tabindex؟

الـ tabindex يحدد إذا كان العنصر قابل للـ focus وبأي ترتيب. قيمته: 0 تخلي العنصر يدخل في الترتيب الطبيعي، وقيم موجبة (1, 2) تحدد ترتيب مخصّص (بس الاستخدام الموجَب نادر لأنه يمكن يربك مستخدمي الكيبورد). -1 يجعل العنصر قابل للـ focus برمجيًا بس مش من Tab.

34- ازاي أتعامل مع عناصر قابلة للتركيز focus و ازاي أتحكم في outline؟

استخدم CSS pseudo-class :focus لتخصيص الستايل عند التركيز. متجنب إزالة تمامًا الـ outline لأن ده يضر مستخدمي لوحة المفاتيح—لو هتشيله قدّم بديل مرئي واضح. مثال جيد: 

:focus { outline: 3px solid #0077cc; }
35- ازاي أقدّم رسائل أخطاء للفورم بطريقة قابلة للوصول (accessible error messages)؟

لو في خطأ، اعرض رسالة قريبة للحقل واستخدم aria-describedby لربط الحقل برسالة الخطأ. لو الرسالة ديناميكية استخدم aria-live="assertive" علشان تقرأها برامج القراءة فورًا. كمان حط الـ focus على أول حقل به خطأ بعد الإرسال لسهولة التصحيح.

100- ايه الفرق بين HTML و CSS و JavaScript؟

HTML مسؤولة عن الهيكل الأساسي (structure) للصفحة: العناوين، الفقرات، الصور، الجداول. CSS مسؤولة عن الشكل والتنسيق (style): الألوان، الخلفيات، الخطوط، توزيع العناصر. JavaScript مسؤولة عن التفاعل (behavior): إنها تخلي الصفحة ديناميكية وتستجيب لتصرفات المستخدم زي الضغط على زرار، التحقق من فورم، أو تحميل بيانات من السيرفر من غير ما تعيد تحميل الصفحة. التلاتة دول مع بعض هما العمود الفقري لأي موقع ويب حديث.

36- ايه الفرق بين GET و POST في الفورم؟

لما تبعت بيانات من الفورم للسيرفر باستخدام attribute method عندك اختيارين أساسيين:

  • GET: البيانات بتتبعت كـ query string في URL (زي ?name=omar&age=25). ده بيظهر في شريط العنوان ويتخزن في history. مناسب للبحث أو استعلامات مش حساسة. لكن غير آمن للبيانات السرية (passwords).
  • POST: البيانات بتتبعت في جسم الطلب (HTTP body)، مش ظاهرة في URL. ده أنسب للمعلومات الحساسة أو العمليات اللي بتغير في قاعدة البيانات.

عمليًا: لو هتعمل search استخدم GET، لو sign up / login / checkout استخدم POST.

37- ايه فائدة attribute action في الفورم؟

action بيحدد العنوان (URL) اللي الفورم هيبعت البيانات ليه بعد ما المستخدم يعمل submit. لو سبته فاضي، هيتبعت لنفس الصفحة. في المواقع الكبيرة غالبًا هتوجه البيانات لملف PHP أو API endpoint. مثال:

<form action="submit.php" method="post">

نصيحة: خليك واضح في تحديد الـ action عشان الكود يبقى مفهوم وسهل الصيانة.

38- ايه الفرق بين button type="submit" و button type="button"؟

لو كتبت <button> من غير type، المتصفح بيفترض إنه submit.

  • submit: بيبعَت الفورم للسيرفر أو يشغّل event submit.
  • button: مش بيعمل أي حاجة افتراضيًا — مفيد لو عايز تكتب JS يتحكم في السلوك (زي فتح modal).
  • reset: بيرجّع كل القيم في الفورم لحالتها الافتراضية.

39- ازاي أستخدم autocomplete في الفورم؟

autocomplete بيساعد المتصفح يملأ الحقول تلقائيًا بناءً على بيانات محفوظة (زي البريد أو العنوان). بتحطه على مستوى <form> أو <input>. مثال:

<input type="email" name="email" autocomplete="email">

ده بيوفر وقت للمستخدم ويحسّن الـ UX. بس لازم تحط قيمة مناسبة (زي name, address, tel...).

40- ايه أهمية enctype في الفورم خصوصًا مع الملفات؟

enctype بتحدد إزاي البيانات هتتشفّر وتتبعَت للسيرفر. في الحالة العادية application/x-www-form-urlencoded. لكن لو عندك input type="file" لازم تستخدم multipart/form-data عشان الملف يتبعت صح.

<form method="post" enctype="multipart/form-data">
41- ازاي أتعامل مع <textarea> وأضبط حجمها؟

<textarea> عنصر لإدخال نصوص طويلة. تقدر تحدد الصفوف rows والأعمدة cols، أو تسيبه وتتحكم بالـ CSS (width, height). تقدر كمان تمنع المستخدم يغيّر الحجم بالـ CSS (resize: none;). مثال:

<textarea rows="5" cols="30"></textarea>
42- ايه الفرق بين checked و selected في الفورم؟

  • checked: للـ radio buttons و checkboxes — يحدد إن الخيار متعلم افتراضيًا.
  • selected: للـ <option> جوا الـ <select> — يحدد الخيار الافتراضي.

43- ازاي أعمل مجموعة radio buttons مرتبطة ببعض؟

الـ radio buttons بيرتبطوا بنفس الـ name. ده يخلي المستخدم يقدر يختار خيار واحد بس من المجموعة: 

<input type="radio" name="gender" value="male"> ذكر <input type="radio" name="gender" value="female"> أنثى
44- ازاي أضيف قائمة اختيار متعددة باستخدام <select multiple>؟

لما تحط attribute multiple في <select> المستخدم يقدر يحدد أكتر من خيار:

<select name="skills" multiple> <option>HTML</option> <option>CSS</option> <option>JavaScript</option> </select>
45- ايه الفرق بين disabled و readonly؟

  • disabled: يمنع التفاعل تمامًا، مش هيتبعت في البيانات.
  • readonly: يمنع التعديل، لكن القيمة هتتبعت للسيرفر.

مثال: input فيه user ID تحطه readonly مش disabled عشان يتبعت.

46- ازاي أستخدم progress و meter في HTML؟

  • <progress>: لعرض تقدم عملية (زي تحميل ملف). مثال: <progress value="70" max="100"></progress>.
  • <meter>: لعرض قيمة داخل نطاق معروف (زي تقييم أو مستوى حرارة). مثال: <meter value="0.7" min="0" max="1"></meter>.

47- ايه أهمية attribute required؟

required بيجبر المستخدم يدخل قيمة قبل إرسال الفورم. لو الحقل فاضي المتصفح هيمنع الإرسال ويظهر رسالة خطأ. لكنه مش بديل للتحقق من السيرفر، لازم تعمل validation هناك برضه.

48- ازاي أستخدم attribute pattern للتحقق من البيانات؟

pattern بيستعمل تعبير منتظم (regex) لتقييد الإدخال. مثال: لقبول بس 5 أرقام:

<input type="text" pattern="\\d{5}">

المتصفح مش هيقبل الإدخال إلا لو وافق الباترن.

49- ازاي أتعامل مع اتجاه النص (RTL أو LTR)؟

HTML بيوفّر attribute dir. ممكن تحطه على <html> أو أي عنصر:

<div dir="rtl">نص عربي</div> <div dir="ltr">English text</div>

كمان تقدر تستخدم CSS direction للتحكم في الاتجاه.

50- ايه الفرق بين lang في <html> وتأثيره؟

Attribute lang بيحدد لغة الصفحة (مثلاً lang="ar"). ده بيساعد محركات البحث تعرف اللغة، وscreen readers تختار نطق مناسب. كمان بعض المتصفحات بتعرض أدوات تدقيق إملائي حسب اللغة.

51- ازاي أكتب تعليقات (comments) في HTML؟

التعليق بيكون كده:

<!-- ده تعليق مش هيظهر في الصفحة -->

التعليقات بتساعدك تفهّم الكود أو تسيب ملاحظات لفريقك. بس تجنّب تحط بيانات حساسة فيها لأنها بتظهر في source code.

52- ايه هو semantic HTML وليه مهم؟

Semantic HTML يعني تستخدم عناصر بمعناها مش بس لشكلها. زي <header>, <nav>, <main>, <footer>, <article>, <section>. ده بيساعد:

  • محركات البحث تفهم المحتوى (SEO).
  • شاشات القارئة تفهم البنية.
  • الكود يبقى أوضح وسهل الصيانة.
53- ايه الفرق بين <section> و <div>؟

<div> عنصر عام (generic) بدون معنى محدد. <section> عنصر semantic بيمثل جزء منطقي من الصفحة ليه عنوان غالبًا. استخدم <section> لو فيه معنى، واستخدم <div> للتجميع أو الستايل لما مفيش عنصر semantic مناسب.

54- امتى استخدم <article> بدل <section>؟

<article> للمحتوى المستقل اللي ممكن يتوزع أو يتفهم لوحده (مقالة، بوست، تعليق). <section> للتقسيمات اللي بتبني الصفحة وممكن تبقى مرتبطة بغيرها. قاعدة بسيطة: لو المحتوى له معنى بذاته — استخدم <article>.

55- ايه أهمية عناصر <header> و <footer>؟

  • <header>: يمثل مقدمة القسم أو الصفحة (شعار، عنوان، روابط).
  • <footer>: يمثل نهاية القسم أو الصفحة (حقوق، روابط إضافية).

ممكن يكون فيه أكتر من header/footer (واحد لكل <article> مثلًا).

56- ايه الفرق بين <div> و <span>؟

الاتنين عناصر عامة (generic containers)، بس الفرق الأساسي إن: - <div> عنصر block-level يعني بياخد السطر بالكامل وبيكسر السطر بعده. - <span> عنصر inline يعني بيظهر جوه النص من غير ما يكسر السطر. مثال: لو عايز تجمع مجموعة عناصر كبيرة في بلوك واحد يبقى تستخدم div، إنما لو عايز تلون كلمة معينة في نص يبقى span أنسب.

57- امتى استخدم <header> بدل <div>؟

<header> عنصر semantic معمول مخصوص عشان يمثل رأس الصفحة أو القسم (زي العنوان الرئيسي + navigation). أما <div> فهو عنصر عام ملّهش معنى semantic. لما تستخدم <header> بتحسن الـ SEO وبتخلي الصفحة أسهل لمحركات البحث ومساعدة لذوي الاحتياجات الخاصة (screen readers).

58- ايه فايدة <footer>؟

<footer> بيستخدم لعرض معلومات ختامية زي حقوق الملكية، روابط سريعة، أو بيانات تواصل. بيقدر يظهر في آخر الصفحة أو في آخر section. استخدامه بيدي معنى أوضح للصفحة وبيخلي هيكلها منظم أكتر.

59- ايه الفرق بين <section> و <article>؟

- <section> بيستخدم عشان تقسيم المحتوى لأجزاء منطقية (زي جزء عن الخدمات، جزء عن الفريق). - <article> بيمثل محتوى مستقل ممكن يتنشر لوحده (زي مقالة أو بوست مدونة). يعني article self-contained بينما section عبارة عن grouping لمحتوى له نفس الهدف.

60- ليه <nav> مهم؟

<nav> بيتحط فيه روابط التنقل الرئيسية للموقع. وجوده بيخلي محركات البحث وأدوات الـ accessibility تعرف إن ده هو navigation الأساسي. من غيره المستخدمين ذوي الإعاقة البصرية ممكن يتلغبطوا في الروابط.

61- ايه الفرق بين inline CSS و internal CSS و external CSS؟

- **Inline CSS**: بيتكتب جوه العنصر نفسه باستخدام الـ style attribute. سريع بس بيخلي الكود ملخبط وصعب الصيانة. - **Internal CSS**: بيتكتب جوه <style> في <head> الصفحة. مفيد لصفحة واحدة بس. - **External CSS**: بيتكتب في ملف خارجي (.css) وبيترابط بالصفحة. أفضل أسلوب عشان يفصل التنسيق عن المحتوى ويسهل إعادة الاستخدام.

62- ازاي بحدد اتجاه النصوص في HTML؟

باستخدام الـ attribute اسمه dir. - dir=\"ltr\": بيخلي النص من الشمال لليمين (زي الإنجليزي). - dir=\"rtl\" : بيخلي النص من اليمين للشمال (زي العربي). كمان ممكن تضيفه في العنصر html كله عشان يطبق على الصفحة كلها.

63- ايه هي character encoding؟

الـ character encoding هي الطريقة اللي بتحدد إزاي الحروف والرموز تتحول لـ bits عشان الكمبيوتر يفهمها. UTF-8 هو الترميز الأكثر استخدامًا لأنه بيدعم معظم لغات العالم بما فيها العربي. عشان كده بنكتب في <head>: <meta charset=\"UTF-8\">.

64- ايه الفرق بين absolute URL و relative URL؟

- **Absolute URL**: بيحتوي على العنوان الكامل للملف (بما في ذلك البروتوكول والدومين). مثال: https://example.com/images/pic.jpg. - **Relative URL**: بيعتمد على موقع الصفحة الحالية. مثال: images/pic.jpg. الـ absolute مفيد لو عايز تشير لملف خارجي أو دومين مختلف، والـ relative أفضل داخل نفس المشروع.

65- ازاي أكتب تعليق في HTML؟

التعليق بيتكتب كده:
<!-- ده تعليق مش بيظهر في الصفحة --> التعليقات بتساعد تشرح الكود لنفسك أو لغيرك وما بتتقرش من المتصفح.

66- ايه الفرق بين <ol> و <ul>؟

- <ol>: Ordered List يعني قايمة مرقمة (1، 2، 3...). - <ul>: Unordered List يعني قايمة نقطية (•). كل عنصر جواهم بيكون باستخدام <li>.

67- ممكن أعمل قايمة جوا قايمة؟

آه، تقدر تعمل Nested Lists. يعني تحط <ul> أو <ol> جوا <li> عشان تعمل قايمة متفرعة. ده مفيد في القوايم الكبيرة زي الـ menu.

68- ايه هو <dl>؟

<dl> هو Definition List. بيستخدم عشان تعمل قايمة بتشرح مصطلحات. - <dt>: المصطلح (term). - <dd>: تعريف المصطلح (description). مثال: قايمة تعريفية بالكلمات أو glossary.

69- ايه فايدة attribute id؟

id بيحدد عنصر معين بشكل فريد في الصفحة. - بيساعدك تعمل CSS أو JavaScript targeting. - كمان بيسمح بإنشاء روابط داخلية للصفحة (anchor links). مهم جدًا إنه يكون unique وما يتكرر.

70- الفرق بين id و class؟

- id: فريد (unique) لكل عنصر. ما ينفعش يتكرر. - class: ممكن يتشارك بين عناصر كتيرة. مثال: لو عندك زرار واحد مخصوص تستخدم id. لو عندك أزرار كتيرة بنفس التنسيق تستخدم class.

71- ايه هو <iframe>؟

<iframe> بيعرض صفحة ويب تانية جوا صفحتك. مثال: تقدر تعرض فيديو YouTube أو خريطة Google Maps. بس خلي بالك الإفراط في استخدامه ممكن يبطّئ الصفحة ويعمل مشاكل في الأمان (security).

72- ازاي أخلي صورة clickable كرابط؟

بتحط الصورة جوا <a> كده:
<a href=\"page.html\"><img src=\"pic.jpg\" alt=\"\"></a> بكده المستخدم لما يضغط على الصورة يروح للرابط.

73- ايه فايدة <meta viewport>؟

<meta name=\"viewport\" content=\"width=device-width, initial-scale=1.0\"> ده بيخلي الصفحة responsive على الموبايل. من غيره الصفحة هتظهر صغيرة ومش متناسبة.

74- ايه الفرق بين block-level و inline elements؟

- **Block-level**: بياخد عرض الصفحة بالكامل ويبدأ في سطر جديد (زي <div>، <p>، <h1>). - **Inline**: بياخد مساحة قد المحتوى بس وما يكسرش السطر (زي <span>، <a>، <img>). ده مهم عشان تفهم سلوك العناصر وتنسقها صح.

75- امتى أستخدم <br>؟

<br> بيعمل line break يعني يكسر السطر. بيتستخدم لما تكون عايز النص يتقسم لسطرين من غير ما تبدأ فقرة جديدة. بس الأفضل تستخدمه قليل، وللفقرات الطويلة الأفضل تستخدم <p>.

76- ايه الفرق بين <em> و <strong>؟

- <em> بيستخدم للتأكيد أو التشديد على كلمة معينة (emphasis)، غالبًا بيتعرض بخط مائل. - <strong> بيستخدم لتأكيد قوي (strong importance)، غالبًا بيتعرض بخط عريض. الفرق مش مجرد شكل، ده كمان دلالي (semantic) لمحركات البحث وأدوات القراءة.

77- ايه الفرق بين <i> و <em>؟

- <i> بيستخدم للخط المائل purely presentation (شكل). - <em> semantic بمعنى بيؤكد على النص في سياق الكلام. بالتالي يفضل تستخدم <em> بدل <i> إلا لو غرضك مجرد الشكل.

78- ايه الفرق بين <b> و <strong>؟

- <b> بيخلي النص عريض من غير معنى semantic. - <strong> بيدي معنى “مهم جدًا” بالإضافة للشكل. يعني <strong> أحسن في الـ SEO والـ accessibility.

79- ايه هو semantic HTML؟

Semantic HTML يعني استخدام عناصر ليها معنى بيوضح الغرض من المحتوى (زي <header>، <footer>، <article>). ده بيساعد محركات البحث والـ screen readers تفهم الصفحة صح، وبيخلي الكود أنظف وأسهل للصيانة.

80- ازاي أكتب جدول في HTML؟

الجدول بيتعمل باستخدام: - <table>: بداية الجدول. - <tr>: صف (table row). - <td>: خلية بيانات (table data). - <th>: خلية عنوان (table header). مثال:
<table> <tr><th>Name</th><th>Age</th></tr> <tr><td>Omar</td><td>25</td></tr> </table>

81- ايه فايدة attribute colspan و rowspan في الجدول؟

- colspan: بيخلي الخلية تمتد على أكتر من عمود. - rowspan: بيخلي الخلية تمتد على أكتر من صف. ده مفيد في الجداول المعقدة زي جداول النتائج أو التقويم.

82- ايه هو <thead> و <tbody> و <tfoot>؟

- <thead>: بيتحط فيه صفوف العناوين. - <tbody>: الصفوف الرئيسية للبيانات. - <tfoot>: صفوف ملخصة (زي إجمالي). التقسيم ده مش بيغير الشكل بس بيخلي الكود أوضح ويدعم الـ accessibility.

83- ازاي أكتب فورمة بسيطة في HTML؟

الفورمة بتبدأ بـ <form>، جواها عناصر إدخال: - <input type=\"text\">: حقل نص. - <input type=\"password\">: كلمة سر. - <input type=\"submit\">: زر إرسال. مثال:
<form action=\"submit.php\" method=\"post\"> <input type=\"text\" name=\"username\"> <input type=\"submit\" value=\"Send\"> </form>

84- ايه الفرق بين method=\"get\" و method=\"post\" في الفورمة؟

- GET: بيبعت البيانات في الرابط (URL). مناسب للبحث، بس مش آمن للبيانات الحساسة. - POST: بيبعت البيانات في body الطلب، مش بتظهر في الرابط. مناسب للتسجيل والدفع. الاختيار بيعتمد على نوع البيانات.

85- ايه هو <label> في الفورمة؟

<label> بيربط النص بحقل الإدخال. لما تضغط على النص المرتبط، بيحدد input تلقائيًا. ده بيسهل على المستخدم ويحسن الـ accessibility. مثال: <label for=\"email\">Email:</label><input id=\"email\" type=\"text\">

86- ايه هو <textarea>؟

<textarea> عنصر لإدخال نصوص طويلة (multi-line). ممكن تحدد الصفوف والأعمدة باستخدام attributes rows و cols. مثال: <textarea rows=\"4\" cols=\"50\"></textarea>

87- ايه الفرق بين <input type=\"radio\"> و <input type=\"checkbox\">؟

- Radio buttons: المستخدم يختار خيار واحد بس من مجموعة. لازم يكونوا بنفس الاسم (name). - Checkboxes: المستخدم يقدر يختار أكتر من خيار في نفس الوقت. مثال: اختيار الجنس Radio، اختيار الهوايات Checkbox.

88- ايه فايدة attribute name في الفورمة؟

name بيحدد اسم الحقل اللي هيتبعت مع البيانات للسيرفر. السيرفر بيستخدم الاسم ده يقرأ القيمة. من غير name، البيانات مش هتتبعت أصلاً.

89- ازاي أعمل قايمة منسدلة؟

باستخدام <select> مع <option>. مثال:
<select name=\"country\"> <option value=\"eg\">Egypt</option> <option value=\"sa\">Saudi Arabia</option> </select>

90- ايه هو placeholder؟

placeholder attribute بيظهر نص داخل input بيختفي أول ما المستخدم يكتب. مثال: <input type=\"text\" placeholder=\"Enter your name\">. مفيد كإرشاد سريع، بس مش بديل عن label.

91- ايه فايدة required attribute؟

required بيخلي الحقل لازم يتملي قبل ما الفورمة تتبعت. المتصفح بيراجع البيانات (client-side validation) ويمنع الإرسال لو الحقل فاضي.

92- ايه الفرق بين <button> و <input type=\"submit\">؟

- <input type=\"submit\">: زرار بسيط للإرسال. نص الزرار بيتحدد بالـ value. - <button>: عنصر أكتر مرونة، ممكن يحتوي على HTML داخلي (نصوص + صور). كمان <button> ليه نوعين: submit أو button عادي.

93- ايه هو <fieldset> و <legend>؟

- <fieldset> بيجمع مجموعة من عناصر الفورمة داخل إطار. - <legend> بيكتب عنوان للإطار ده. مثال: بيانات شخصية (fieldset) فيها الاسم، الإيميل، والتليفون.

94- ايه فايدة autocomplete attribute؟

autocomplete بيخلي المتصفح يقترح بيانات سابقة للمستخدم (زي الإيميل أو الاسم). ممكن يكون on أو off. مثال: <input type=\"email\" autocomplete=\"on\">

95- ايه الفرق بين inline frame (iframe) و embed؟

- <iframe>: بيعرض صفحة ويب كاملة جوا صفحتك. - <embed>: بيستخدم لإضافة محتوى خارجي زي فيديو أو ملف PDF أو Flash. الاتنين بيضيفوا محتوى خارجي، بس iframe مخصص أكتر للصفحات، و embed للميديا.

96- ايه الفرق بين <script> في <head> و في <body>؟

- لما تحط <script> في <head>، المتصفح بيوقف تحميل الصفحة لحد ما يحمّل الكود، وده ممكن يبطّئ عرض المحتوى. - لما تحطه قبل </body>، الصفحة كلها بتظهر الأول وبعدها يتنفذ السكربت. عشان كده يفضل دايمًا تحط JavaScript قبل </body> أو تستخدم attribute زي defer أو async.

97- ايه الفرق بين defer و async في <script>؟

- **async**: بيحمّل السكربت في الخلفية ويتنفذ أول ما يخلص التحميل، حتى لو الصفحة لسه بتتحمّل. - **defer**: بيحمّل السكربت في الخلفية لكنه ما يتنفذش غير بعد ما الصفحة تخلص تحميل بالكامل. غالبًا defer أفضل للكود اللي بيعتمد على عناصر الصفحة.

98- ايه هو <noscript>؟

<noscript> بيُعرض للمستخدمين اللي متصفحهم مش بيدعم JavaScript أو اللي موقفينه. ممكن تكتب فيه رسالة أو روابط بديلة عشان الصفحة تفضل مفهومة حتى من غير JS.

99- ايه الفرق بين <link> و <a>؟

- <link>: بيتكتب في <head> لربط الصفحة بملفات خارجية زي CSS أو أي موارد تانية. مش عنصر مرئي. - <a>: بيربط المستخدم بصفحة أو ملف تاني، وده عنصر مرئي بيتضغط عليه. يعني link للموارد التقنية و a للتنقل بين الصفحات.

100- ايه هو <base>؟

<base> بيتكتب في <head> ويحدد عنوان أساسي (base URL) للروابط النسبية في الصفحة. مثال: لو كتبت <base href=\"https://example.com/\">، أي رابط نسبي زي images/pic.jpg هيتحول لـ https://example.com/images/pic.jpg.

101- ازاي أضيف أيقونة للصفحة (favicon)؟

بتحط الكود في <head>:
<link rel=\"icon\" href=\"favicon.ico\" type=\"image/x-icon\"> الأيقونة دي بتظهر في التاب بتاع المتصفح والبوكماركس.

102- ايه فايدة attribute target في <a>؟

target بيحدد مكان فتح الرابط: - _self: في نفس التاب (ده الافتراضي). - _blank: في تاب جديد. - _parent و _top: حالات خاصة بالإطارات (frames). أكتر حاجة مستخدمة النهارده _blank عشان يفتح في نافذة جديدة.

103- ايه الفرق بين relative path و absolute path للصور؟

- relative path: بيعتمد على مكان الملف الحالي. مثال: images/pic.jpg. - absolute path: بيكتب المسار الكامل من أول الجذر (أو الـ URL). مثال: /assets/images/pic.jpg أو https://example.com/pic.jpg. relative مفيد للمشاريع الداخلية، absolute مفيد لو الصور جايه من سيرفر تاني.

104- ازاي أخلي عنصر يظهر أو يختفي بس باستخدام HTML؟

باستخدام attribute اسمه hidden. مثال: <p hidden>النص ده مش هيظهر</p>. المتصفح بيتجاهله في العرض لكنه موجود في الكود.

105- ايه فايدة download attribute في <a>؟

download attribute بيخلي الرابط ينزل الملف بدل ما يفتحه. مثال: <a href=\"file.pdf\" download>Download File</a>. كده المستخدم هيحمّل الملف مباشرة.

106- ايه الفرق بين SVG و IMG؟

- IMG: بيعرض صورة نقطية (bitmap) زي PNG أو JPG. الجودة بتتأثر بالتكبير. - SVG: رسمة متجهة (vector)، بتتحافظ على الجودة مهما كبرت. SVG مناسب للأيقونات والشعارات، IMG للصور العادية.

107- ايه هو <figure> و <figcaption>؟

- <figure>: بيستخدم لاحتواء صورة أو رسم أو أي محتوى مرئي. - <figcaption>: تعليق أو وصف للمحتوى داخل figure. مثال: صورة + شرحها النصي.

108- ازاي أضيف فيديو في HTML؟

باستخدام <video> element:
<video controls> <source src=\"movie.mp4\" type=\"video/mp4\"> المتصفح مش بيدعم الفيديو ده. </video> ممكن تضيف attributes زي autoplay, loop, muted.

109- ازاي أضيف صوت في HTML؟

باستخدام <audio> element:
<audio controls> <source src=\"sound.mp3\" type=\"audio/mpeg\"> المتصفح مش بيدعم الصوت ده. </audio>

110- ايه فايدة attribute controls في <audio> و <video>؟

controls attribute بيظهر أزرار التحكم (تشغيل، إيقاف، صوت) للمستخدم. من غيره، المستخدم مش هيقدر يتحكم في تشغيل الملف.

111- ايه الفرق بين HTML5 و الإصدارات القديمة؟

HTML5 جاب معاه عناصر semantic جديدة (<header>, <footer>, <article>). كمان أضاف دعم للفيديو والصوت بدون إضافات خارجية. دعم Local Storage, Canvas, Geolocation. يعني بقى أقوى وأسهل في تطوير التطبيقات الحديثة.

112- ايه هو <canvas>؟

<canvas> عنصر بيخليك ترسم جرافيكس باستخدام JavaScript. مفيد للألعاب، الرسومات التفاعلية، charts. هو مجرد مساحة فارغة، ولازم تستخدم JS لرسم أي حاجة عليه.

113- ازاي أعمل input يقبل ألوان؟

باستخدام <input type=\"color\">. ده بيفتح أداة اختيار الألوان في المتصفح. القيمة اللي بترجع بتكون كـ hex code (#ff0000 مثلًا).

114- ازاي أخلي input يقبل تواريخ؟

باستخدام <input type=\"date\">. بيفتح calendar picker في المتصفح عشان المستخدم يختار تاريخ. القيمة اللي بترجع بتكون بصيغة yyyy-mm-dd.

115- ايه الفرق بين HTML و CSS و JavaScript؟

- HTML: بيحدد هيكل الصفحة والعناصر (العناوين، النصوص، الصور، الفورمات). - CSS: مسؤول عن الشكل والتنسيق (ألوان، خطوط، مسافات، layout). - JavaScript: بيضيف التفاعل والديناميكية (الأزرار، الحركات، تحديث البيانات بدون refresh). التلاتة مع بعض هما الأساس لأي موقع ويب حديث.
